折腾了几天总算从学校网站上把课程表抓取下来了，在这记录一下过程中遇到的各种问题

OkHttp的一些问题

　　在一开始就遇到了问题，直接请求学校网站时总是获取不到内容，后来才发现是学校网站证书的问题，以下方法可以简单的让okhttp信任所有证书，也可以设置为信任某一证书，不过过程更复杂一些。

　　okhttp默认是自动加载重定向的网站的，而且可以设定自动记录Cookie。一开始一直在手动保存Cookie并添加，后来才发现还有这个功能

模拟登陆过程

　　用的抓包工具HttpWatch分析登陆过程中用到的参数，对于密码账号验证码等参数，都比较好搞定。
　　其中的一些变化的参数纠结了好久,最后才在登陆页面的html文件的input标签下找到的，利用Jsoup可以从中解析出值。

　　登录请求的url中跟着一个jsessionID值也是在html文件的一个标签下的action属性中找到

在发送请求过程中应在请求中加入如下头部模拟浏览器，服务器才会返回数据。
　　addHeader("User-Agent","Mozilla/5.0 (Windows NT 6.3; WOW64; Trident/7.0; LCJB; rv:11.0) like Gecko")

　　最后登录了学校网站页面后，在请求课程表页面时，又遇到了很多问题，由于课表所在的位置与学校信息网站的位置没有在同一个网站上，其中会偶远程调用过程，在请求课表时会先发送一个post请求，其中的部分数据解析出来后，有一个scrptsessionId的参数一直找不到，后来就硬着头皮去看其中的脚本代码，终于在一个.js的脚本文件中找到，不过在发送请求后，服务器返回显示远程调用有一个参数不对。又陷入了长时间的纠结。
　　最后发现如果跳过登录直接请求课表的那个网站会返回一个登录页面，按照正常的登录过程登陆后直接返回的就是课表的那个网站，总算是把课表抓下来了。
　　虽然按照正常的流程没能成功，但在过程中对浏览器的工作过程以及脚本程序有了一定的理解。对于html中的脚本程序也多了一些了解．
